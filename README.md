# Бронирование ресторанов для мероприятий

Платформа для бронирования ресторанов на свадьбы, юбилеи, корпоративы и другие мероприятия. Владельцы ресторанов могут размещать свои заведения и управлять доступными датами, а клиенты могут просматривать и бронировать подходящие места.

## Основной функционал

### 🔐 Система аутентификации
- Регистрация с выбором роли (клиент или владелец ресторана)
- Вход в систему с email и паролем
- Безопасное хранение паролей (bcrypt)
- NextAuth v5 интеграция
- Защищенные маршруты по ролям

### 👥 Роли пользователей
- **CUSTOMER** - клиенты, которые бронируют рестораны
- **RESTAURANT_OWNER** - владельцы ресторанов
- **ADMIN** - администраторы системы

## Функционал для клиентов

### 🏠 Главная страница
- Просмотр всех доступных ресторанов
- Фильтрация по регионам (Ташкент, Самарканд, Бухара)
- Отображение количества найденных ресторанов
- Поисковая форма с выбором региона
- Карточки ресторанов с основной информацией:
  - Название и фото
  - Рейтинг (⭐)
  - Город и регион
  - Описание
  - Вместимость (количество гостей)
  - Цена за час аренды

### 🏢 Страница ресторана
- Подробная информация о ресторане
- Галерея фотографий
- Характеристики:
  - Вместимость
  - Цена за час
  - Адрес и контакты
  - Регион
- Интерактивный календарь с доступными датами
- Визуальное выделение дат со свободными слотами
- Просмотр доступных временных слотов на выбранную дату
- Информация о каждом слоте:
  - Время начала и окончания
  - Тип мероприятия (свадьба, юбилей, корпоратив, другое)
  - Цена
  - Статус (доступен/забронирован)

### 📅 Бронирование
- Выбор даты в календаре
- Выбор временного слота
- Форма бронирования:
  - Имя клиента
  - Номер телефона
  - Email
  - Количество гостей
  - Тип мероприятия
  - Дополнительные пожелания
- Мгновенное создание бронирования
- Автоматическая проверка доступности слота

### 📋 Мои бронирования
- Список всех бронирований пользователя
- Статусы бронирований:
  - Ожидает подтверждения (PENDING)
  - Подтверждено (CONFIRMED)
  - Отменено (CANCELLED)
  - Завершено (COMPLETED)
- Информация о каждом бронировании:
  - Название ресторана (с ссылкой)
  - Дата и время
  - Количество гостей
  - Общая стоимость
  - Тип мероприятия
  - Текущий статус
- Кнопка отмены бронирования
- Сортировка по дате (новые первыми)

## Функционал для владельцев ресторанов

### 📊 Dashboard
- Общая статистика:
  - Количество ресторанов владельца
  - Общее количество бронирований
  - Бронирования в ожидании
  - Подтвержденные бронирования
- Быстрые ссылки:
  - Мои рестораны
  - Добавить ресторан
  - Все бронирования

### 🏢 Мои рестораны
- Список всех ресторанов владельца
- **Важно**: Владелец видит только СВОИ рестораны
- Для каждого ресторана отображается:
  - Основное фото
  - Название
  - Регион и город
  - Вместимость
  - Цена за час
  - Статус (активен/неактивен)
  - Количество бронирований
  - Дата создания
- Кнопки действий:
  - Редактировать
  - Управление слотами
  - Просмотр бронирований
  - Удалить ресторан

### ➕ Создание ресторана
- Форма добавления нового ресторана:
  - Название
  - Описание (расширенный редактор текста)
  - Регион (выбор из списка)
  - Город
  - Адрес
  - Вместимость (количество гостей)
  - Цена за час аренды
  - Номер телефона
  - Email
- Автоматическое создание slug из названия
- Привязка ресторана к текущему пользователю
- Перенаправление на страницу управления после создания

### ✏️ Редактирование ресторана
- Форма с предзаполненными данными
- Возможность изменить:
  - Название
  - Описание
  - Регион
  - Город
  - Адрес
  - Вместимость
  - Цену
  - Контакты
  - Статус (активен/неактивен)
- **Защита**: Только владелец может редактировать свой ресторан
- Проверка прав доступа на уровне сервера

### 🗑️ Удаление ресторана
- Кнопка "Удалить ресторан" на странице управления
- Модальное окно с подтверждением
- Предупреждение о необратимости действия
- Каскадное удаление:
  - Удаляются все связанные слоты
  - Удаляются все бронирования
  - Удаляются все отзывы
  - Удаляются все фотографии
- **Защита**: Только владелец может удалить свой ресторан

### 🕒 Управление временными слотами
- Страница управления доступными датами
- Две формы создания:

  **Форма создания одного слота:**
  - Выбор даты
  - Время начала
  - Время окончания
  - Тип мероприятия
  - Цена (по умолчанию из настроек ресторана)

  **Массовое создание слотов:**
  - Дата начала периода
  - Дата окончания периода
  - Время начала
  - Время окончания
  - Тип мероприятия
  - Цена
  - Создание слотов на каждый день указанного периода

- Список всех слотов с группировкой по датам
- Для каждого слота:
  - Дата
  - Время
  - Тип мероприятия
  - Цена
  - Статус (доступен/забронирован)
  - Кнопка удаления (только для свободных слотов)

### 📋 Просмотр бронирований
- Все бронирования для конкретного ресторана
- Фильтрация по статусам
- Управление статусами бронирований:
  - Подтверждение бронирования
  - Отмена бронирования
  - Отметка как завершенное
- Контактная информация клиентов
- Детали каждого бронирования

## Структура базы данных

### Модели

#### User (Пользователи)
- id, email, password (хешированный)
- role: CUSTOMER | RESTAURANT_OWNER | ADMIN
- name, phone
- связи: restaurants[], bookings[], reviews[]

#### Region (Регионы)
- id, name, slug
- связи: restaurants[]
- Предустановленные: Ташкент, Самарканд, Бухара

#### Restaurant (Рестораны)
- id, name, slug
- description, address, city
- capacity (вместимость), pricePerHour
- ownerId → User (владелец)
- regionId → Region
- phone, email
- isActive (статус)
- связи: images[], eventSlots[], bookings[], reviews[]

#### RestaurantImage (Фотографии)
- id, url
- isPrimary (основное фото)
- restaurantId → Restaurant

#### EventSlot (Временные слоты)
- id, date, startTime, endTime
- eventType: WEDDING | BIRTHDAY | CORPORATE | OTHER
- price
- isBooked (забронирован или нет)
- restaurantId → Restaurant
- связи: bookings[]

#### Booking (Бронирования)
- id, bookingDate
- guestCount, customerName, customerPhone, customerEmail
- eventType
- totalPrice
- status: PENDING | CONFIRMED | CANCELLED | COMPLETED
- specialRequests (особые пожелания)
- userId → User
- restaurantId → Restaurant
- slotId → EventSlot

#### Review (Отзывы)
- id, rating (1-5), comment
- userId → User
- restaurantId → Restaurant
- createdAt

## Установка и запуск

### Предварительные требования
- Node.js 18+
- PostgreSQL 14+
- npm или yarn

### Шаги установки

1. **Клонировать репозиторий**
```bash
git clone <repository-url>
cd kitchen
```

2. **Установить зависимости**
```bash
npm install
```

3. **Настроить переменные окружения**
Создать файл `.env`:
```env
# Database
DATABASE_URL="postgresql://postgres:nextjs@localhost:5432/tatarkitchen?schema=public"

# Auth
AUTH_SECRET="your-secret-key-here"
```

4. **Запустить PostgreSQL**
```bash
pg_ctl -D "C:\ProgramData\scoop\apps\postgresql\current\data" start
```

5. **Применить миграции**
```bash
npx prisma migrate dev --name init
```

6. **Заполнить базу тестовыми данными**
```bash
npx prisma db seed
```

7. **Запустить проект**
```bash
npm run dev
```

Приложение будет доступно по адресу: [http://localhost:3000](http://localhost:3000)

## Безопасность

### Защита данных
- ✅ Все пароли хешируются с использованием bcrypt
- ✅ Защищенные маршруты с проверкой аутентификации
- ✅ Роль-основанный контроль доступа (RBAC)
- ✅ Владельцы видят только свои рестораны
- ✅ Проверка владельца перед редактированием/удалением
- ✅ Server Actions с валидацией на сервере
- ✅ Защита от SQL-инъекций (Prisma)
- ✅ Zod валидация всех входных данных

### Проверки на уровне Server Actions
- `getMyRestaurants(userId)` - возвращает только рестораны владельца
- `updateRestaurant(id, data, userId)` - проверяет `ownerId === userId`
- `deleteRestaurant(id, userId)` - проверяет `ownerId === userId`
- `createBooking()` - проверяет доступность слота
- `deleteEventSlot()` - запрещает удаление забронированных слотов
